## Codebase Patterns
- Branch: ralph/general-browser-agent (this repo was bare; init'd on first iteration).
- TS: ESM only. tsconfig has `isolatedModules: true`, so a stub file with no import/export
  must include `export {};` to be treated as a module (else top-level `const` collides
  across files in global scope).
- TS: target ES2023, module ES2022, moduleResolution `bundler`. Use `.js` extensions in
  relative imports (TS rewrites them at build, tsx accepts them at run).
- TS tests: `node --test --import tsx harness/ts/tests/*.test.ts` (no jest, no vitest).
- TS strict mode includes `noUncheckedIndexedAccess`: `arr[0]` is `T | undefined`.
  Cast at the boundary (`arr[0] as string`) or guard with `if (!x) ...`.
- CDP: do NOT pull in `chrome-remote-interface` or `puppeteer`. Node 22+ has global
  WebSocket and fetch — `harness/ts/cdp/{launch,client}.ts` is enough. Chrome is at
  `/usr/bin/google-chrome` (143). Override with `GBA_CHROME_PATH`.
- Chrome launch: `--remote-debugging-port=0` (random port), parse stderr for
  `DevTools listening on ws://...:PORT/`. Always pass `--user-data-dir=<temp>` and rm
  it on close. Headless flag is `--headless=new`.
- Process hygiene: every Chrome handle MUST expose a `close()` that SIGTERM-then-SIGKILLs
  the child and `rm -rf`s the user-data-dir. Tests assert this.
- Agent contract (US-002): `harness/ts/agent/{types,agent,trajectory,browser_session,python_bridge}.ts`.
  Every agent extends `Agent`, owns its `Trajectory`, opens it via
  `Trajectory.open({runsRoot,agent,task,seed},{agent_id,task_id,seed})`,
  and ends with `finish({terminal_state, verifier_verdict?, decline_reason?})`.
  Trajectories are JSONL `meta`/`step`/`end` lines, gzipped to `trajectory.jsonl.gz`
  on finish. The .gz is the done-marker for the resumable runner (US-010).
- Cross-language agents: TS spawns `python -m gba_agent.runner --agent <path>` via
  `PythonAgentBridge` (`harness/ts/agent/python_bridge.ts`). Protocol is line-delimited
  JSON-RPC 2.0 over stdio; `runPythonAgent(...)` registers browser/budget/trajectory
  RPC handlers and awaits a single `agent.run` call. Python proxies live in
  `harness/python/gba_agent/` and use stdlib only.
- Python: `harness/python/gba_agent/` is on PYTHONPATH via pyproject's
  `[tool.pytest.ini_options].pythonpath` for tests, and via env var set by
  `PythonAgentBridge.spawn` for the runner. No pip install needed.
- Python rpc.py: inbound RPC handlers MUST run on a worker thread so they may
  issue their own `rpc.call(...)` back to the peer without deadlocking the
  reader. Already handled inside `_dispatch_request`.
- Tests: `make test` now runs both `npm test` (TS) and `uv run pytest` (Python).
  TS tests under `harness/ts/tests/*.test.ts`; Python tests under
  `harness/python/tests/test_*.py`.
- BrowserSession.evaluate: `Runtime.evaluate` returns a result whose shape can
  contain `exceptionDetails` if the page expression throws. Always check that and
  re-raise; otherwise the caller silently sees `undefined` for thrown evaluations.
- Makefile: `make install` runs `npm install` + `uv sync`. `make smoke` is wired,
  `make test` is wired (US-002). eval/tournament/report still stubs (US-009..U-011).
- BrowserPool (US-003) at `harness/ts/cdp/pool.ts`. Use `BrowserPool.create({size})`
  (default 4 / GBA_POOL_SIZE) then `pool.acquire()` / `pool.release(s)` or
  `pool.withSession(fn, {taskTimeoutMs})`. Each acquire/release destroys-and-
  respawns the slot so isolation is the contract (cookies/localStorage/cache
  are guaranteed fresh). Wall-clock deadline throws `SessionTimeoutError`
  out of session methods; agents must catch it and finish trajectories with
  `terminal_state="SESSION_TIMEOUT"` (see agents/CLAUDE.md).
- Chrome is spawned `detached:true` (own pgid) and killed via process group
  (`process.kill(-pid, sig)`) so renderer / GPU / network subprocesses die
  before `rm -rf` the user-data-dir. Without this, rm races ENOTEMPTY/EBUSY.
- snapshot/restore on PooledBrowserSession captures URL + cookies (Network.
  getAllCookies) + local/sessionStorage; restore clears, sets cookies, navigates,
  then replays storage. Pass {includeMhtml:true} for an audit DOM payload via
  Page.captureSnapshot — MHTML is NOT replayed by restore (audit only).
- Pool internals are stored in a module-level WeakMap keyed by session.
  release() does NOT delete the WeakMap entry: safeCdp() needs to read
  `deadlineHit` when the WS-close event fires AFTER release returns, so it
  can convert a "CDP session closed" rejection into SessionTimeoutError.
  Once the agent drops its session reference, GC reclaims the entry.
- LLMClient (US-004) at `harness/ts/llm/`. Construct via `defaultClient({budget,
  trajectory, paradigmSeed})` to pick up OPENAI_API_KEY / GEMINI_API_KEY from
  the env. `await llm.call(model, messages, opts)` routes by model prefix
  (gpt-*/o4-*/o3-* → OpenAI; gemini-* → Gemini). Cache lives at
  runs/.cache/llm/<key[:2]>/<key>.json keyed by sha256(stable-json({model,
  messages, opts, paradigm_seed})). mode='record' (default) calls on miss
  and persists; mode='replay' throws LLMReplayMissError on miss.
- LLM redaction is the LLMClient's job, not the provider's. Providers throw
  raw Errors that may include the API key (Gemini's URL has it as a query
  param). LLMClient catches at the boundary, replaces every `redactValues`
  occurrence with `[REDACTED]`, and rethrows. Pass `redactValues: [apiKey]`
  to LLMClient — defaultClient() does this automatically.
- Trajectory JSONL gained a third line kind: `{"kind":"llm_call", model,
  prompt_hash, prompt_tokens, completion_tokens, latency_ms, cost_usd,
  cached}`. Recorded via `trajectory.recordLlmCall(...)` from inside
  LLMClient. Snapshot accessor: `trajectory.snapshotLlmCalls()`.
- Python agents reach the LLM via `gba_agent.LLMClient(rpc)` which forwards
  to TS over the existing bridge. Register the handler by passing
  `{llm: client}` to `runPythonAgent`. The Python proxy translates known
  TS errors (LLMReplayMissError, LLMProviderUnavailableError) back to
  typed Python exceptions (LLMReplayMiss, LLMProviderUnavailable).
- Verifier framework (US-005) at `harness/ts/verifier/`. Three kinds:
  `js` (CDP Runtime.evaluate in page, awaitPromise=true), `trajectory_predicate`
  (JS expression run in Node against `{steps, llmCalls, metadata}` snapshots),
  `llm_judge` (temp=0, n=3 majority vote; only allowed when task tagged
  `judge_required`). `loadTaskFile(path)` parses YAML and validates the
  verifier spec at load time — malformed specs (missing fields, bad kind,
  llm_judge without judge_required, predicate that won't compile) fail
  before the tournament starts. `verify(task, ctx, opts)` dispatches the
  right verifier impl, optionally appends a `verification` JSONL line to
  the open trajectory, and writes a `verdict.json` sidecar in the
  trajectory directory.
- Trajectory JSONL gained a fourth line kind: `{"kind":"verification",
  pass, score, reason, verifier_kind, verified_at}`. `finish()` now folds
  the most-recent verification into `metadata.verifier_verdict` when no
  explicit verdict is passed. Snapshot accessor:
  `trajectory.snapshotVerifications()`.
- Minimal YAML loader at `harness/ts/verifier/yaml.ts` (no js-yaml dep).
  Supports the subset task specs need: scalar mappings, nested mappings,
  block + flow lists, quoted strings, comments, block scalars (| > with
  -+ chomp). Tabs in indent rejected. Inline mappings inside list elements
  are explicitly unsupported (workaround: nest the mapping). When US-006
  fixtures push the format past this subset, swap in js-yaml — but the
  subset has been chosen to cover the real task specs in this repo.
- Hostile fixtures (US-006) at `tasks/fixtures/{server,serve}.ts` +
  `tasks/fixtures/pages/*.ts`. Server is plain `node:http` (no Express
  dep). Each page is a single exported HTML constant; route + (optional)
  state endpoints live in server.ts. YAML task specs at
  `tasks/suite/hard/*.yaml` use `start_url: "fixtures://<route>"` —
  `resolveFixtureUrl(start_url, server.origin)` rewrites it to the live
  127.0.0.1:<random> URL at run time. Use `make fixtures` to boot the
  server in the foreground for manual poking. See `tasks/CLAUDE.md` for
  the recipe for adding a new fixture (US-007/US-008 will extend this).
- Eval runner (US-006 partial; full US-010 supersedes) at
  `harness/ts/eval/runner.ts`. `runEval({agent, slice, seeds, runsRoot})`
  loads every yaml under `tasks/suite/<slice>/`, conditionally starts
  the fixtures server when any task is `fixtures://`, instantiates the
  agent (loaded by file path from `agents/<id>/agent.ts`; built-in alias
  `trivial → click-first-link` lives in `AGENT_ALIASES`), runs each
  (task, seed) cell, calls `verify()`, and prints a summary table. The
  tournament-grade resumable runner with leaderboard JSON lands in US-010
  — when adding it, REPLACE the eval CLI body, do NOT add a parallel
  entry point. Per-difficulty budgets live in `DIFFICULTY_BUDGETS` and
  must stay in sync with US-010's AC #2.
- Verifier `js` expressions can hit fixtures-server endpoints via
  `fetch('/__feature/last')` because the page origin === the fixtures
  origin during a run. `awaitPromise=true` on Runtime.evaluate (set in
  JsVerifier) means the verifier may return a Promise<bool>. Use this
  instead of stuffing state into `window.__test` whenever the relevant
  state is server-side.
- Multi-page fixtures (US-007 iframe-drag): each child frame is a
  separate route exporting its own `*_HTML` constant. Children relay
  state via `window.parent.postMessage(...)` to a listener wired in
  the parent's IIFE that maintains the canonical `window.__test`. The
  parent's listener IS the source of truth for the verifier — children
  never write `window.__test` themselves. postMessage delivery is
  asynchronous (microtask/task), so cheats and any agent must poll
  parent's `__test` for a few frames before asserting.
- Trivial-fails / cheat-passes pattern scales to richer hostile pages:
  the cheat closure is just an inline `browser.evaluate(...)` snippet.
  For server-receipt fixtures (shadow-form, conditional-form), use a
  `MutationObserver` on `<title>` (set to "submitted" on success) plus
  a setTimeout fallback so the cheat blocks deterministically until
  the server ack lands. For postMessage fixtures, poll the parent's
  state with `await new Promise(r=>setTimeout(r,30))` until the
  recorded entry appears.

- **Learnings for future iterations:**
  - WeakMap entries indexed by Session objects MUST NOT be deleted in
    release(). The pool's safeCdp() runs LATER (after the WS close event
    fires from chrome's exit, on a subsequent I/O tick) and needs to read
    `deadlineHit` to wrap "CDP session closed" into SessionTimeoutError.
    Synchronously deleting in release() causes a race that downgrades the
    error to a generic Error. GC reclaims the entry once the agent drops
    its session reference — no leak.
  - Chrome MUST be spawned `detached:true` so we can SIGKILL its whole
    process group (`process.kill(-pid, signal)`). Without this, the
    renderer/GPU/network subprocesses keep writing into --user-data-dir
    after the parent dies and `rm -rf` races with ENOTEMPTY / EBUSY. This
    fixed both the new pool test AND a pre-existing flake in
    smoke.test.ts ("temp user-data-dir is removed on close").
  - When the parent chrome is already dead (e.g. external SIGKILL in the
    crash-replace test), still call killGroup(SIGKILL) before rm — group
    members survive the leader. killChild() now does this.
  - Acquire/release-then-respawn is the only reliable way to guarantee FR-4
    isolation without a per-slot scrub script that future agents would
    forget. ~1s spawn cost is acceptable for the research harness.
  - `Network.enable` MUST be sent at slot init for snapshot/restore (cookie
    domain methods are gated on it). Page.enable alone is not enough.
  - Cookies don't persist on data: URLs (opaque origin), so any pool
    snapshot/restore test needs an http origin. The test file spins up a
    tiny http.createServer per test on 127.0.0.1:0 and tears it down at
    the end. Pattern: `startFixtureServer()` returning {origin,url(),close()}.
  - `assert.rejects(() => stat(dir), /ENOENT/)` is the right assertion for
    "this directory was removed". stat() throws an Error whose .message
    starts with "ENOENT: no such file or directory, stat '...'" — the
    regex matches.
  - rm({maxRetries:5, retryDelay:100}) is a useful safety net even with
    pgid-killing — keep it. Linear backoff on EBUSY/ENOTEMPTY/EMFILE/etc.
  - Pool wakeWaiter() runs in a tight while-loop pairing waiters with
    idle slots; safe because both arrays mutate synchronously inside the
    loop. Don't add awaits inside that loop or you'll get double-wakes.
  - PooledBrowserSession exposes `userDataDir` and `pid` getters (read
    through the WeakMap). Useful for tests; agents shouldn't need them
    in normal operation but they don't leak anything sensitive.
  - The hang-agent test had to await its inner trajectory-write Promise
    AFTER the outer `pool.withSession` race rejected — otherwise the .gz
    file might still be in flight when the test asserts on it. Pattern
    captured in `runHangAgent` helper inside pool.test.ts.

  - The `gba_agent` Python `__init__.py` re-exports the public surface;
    when adding a new module (llm.py here), add its symbols to both the
    `from .x import ...` and `__all__`. Test imports use `from gba_agent
    import LLMClient` so this is load-bearing.

    cells can share replay data. Putting it under `runs/.cache/llm/`
    means `make clean` (which `rm -rf runs/`) wipes it alongside
    trajectories, but a partial `rm -rf runs/<agent>/` won't.
  - `noUncheckedIndexedAccess` plus stable-key extraction means stripping
    `paradigm_seed` from opts uses the rename pattern: `const {
    paradigm_seed: _drop, ...rest } = opts;`. The `_drop` underscore is
    necessary or lint complains about an unused binding.
  - When adding a JSON-RPC handler, pick names with a dotted namespace
    prefix (browser.*, budget.*, trajectory.*, llm.*). The Python proxy
    raises typed exceptions by string-matching the TS error name in the
    RPC error message — not pretty but stable enough for now.

    `contents[]`; they go on a top-level `systemInstruction` field. The
    provider extracts and joins them with "\n\n" before constructing
    contents. Forgetting this gives a 400 about "invalid role: system".

    so editing pricing.ts retroactively updates every replayed run's
    cost. Don't bake `cost_usd` into the cache file.

    the cache.set() above it already persisted the response. That's
    intentional — the call already cost money/tokens, the agent just
    can't make any MORE calls. A replay with a fresh budget will pick
    up the cached response.
  - For Python tests, paired StdioJsonRpc peers over an `os.pipe()` pair
    is a clean way to exercise RPC round trips without subprocesses or
    network. Pattern: `_pair()` returns (a, b); register handlers on b,
    issue calls from a, both `start()`-ed.
  - We DO NOT redact at the provider layer. The provider's contract is
    "throw any error verbatim"; LLMClient redacts at its boundary. This
    keeps providers simple and lets tests assert that the unredacted
    error contains the status code without bothering about which keys
    are configured.

  - `Trajectory.recordVerification` MUST be called before `finish()`
    because `finish()` ends the writer and gzips. The verifier framework
    catches this for you (it skips the trajectory write when
    `trajectory.isFinished`), but agents writing self-checks mid-run
    need to be explicit.
  - The `verdict.json` sidecar is for fast inspection without opening the
    `.gz`. Both writes happen by default; if you only want one, pass
    `{writeAuditFile:false}` or `{recordIntoTrajectory:false}`.
  - The minimal YAML loader is intentionally narrow. If you hit
    `InvalidYamlError` while writing a fixture spec for US-006/008/009,
    rewrite the spec to fit the subset (e.g. nest a mapping instead of
    inlining `- a: 1`) rather than reaching for js-yaml — keeping the
    parser tiny is part of the no-deps contract.
  - For llm_judge, three calls with distinct `paradigm_seed` values
    populate three independent cache entries. Replay is therefore
    deterministic across reruns even though the model is sampled three
    times. Raising N (n=5, n=7) is a one-line change in `llm_judge.ts`.
  - Verifier `js` expression is wrapped as `(async () => (EXPR))()` then
    sent with `awaitPromise=true`. This lets fixture authors return a
    Promise (e.g. `fetch('/__test/last').then(j => j.user === 'alice')`)
    without thinking about async vs sync. Bare booleans, objects, and
    Promises all work.
  - The verifier-side trivial-fails / cheat-passes test pattern: spin up
    a `node:http` fixture server (same pattern as pool tests), navigate
    each agent on a fresh `CdpBrowserSession` (so trivial agent's
    navigation away doesn't taint the cheat agent's run), call `verify()`
    against the still-open browser session, assert the verdict.
  - Trivial agent's `terminal_state` was DONE for the failing case — it
    thought it succeeded, but the verifier disagreed. This is the
    intended distinction: an agent's self-reported terminal state is
    independent of the verifier verdict, and tournament metrics will use
    the verifier verdict as ground truth.
